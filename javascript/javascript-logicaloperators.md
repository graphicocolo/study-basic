# 論理演算子：`||` / `??` / `&&`

> 3つは似ているようで「右を使う条件」が違う。

---

## まず「偽っぽい値（falsy）」を知っておく

JavaScript には「偽っぽい」と判断される値がある：

```
false, 0, '', null, undefined, NaN
```

```[]``` や ```{}``` は、truthy として判定される

---

## `||`「左がダメなら右を使う」（論理和）

「ダメ」= falsy な値（上のリスト全部）

```js
'保存済みテーマ' || 'システムテーマ'  // → '保存済みテーマ'（左がOKなのでそのまま）
''              || 'システムテーマ'  // → 'システムテーマ'（空文字はダメ扱い）
0               || 'システムテーマ'  // → 'システムテーマ'（0 もダメ扱い）
```

> 「左の人が答えられなかったら、右の人に答えてもらう。ただし、0点も無言扱い。」

左辺が true の場合は左辺を返し、右辺を評価しない

```tsx
// 値が falsy なら 0 にする
const minutes = parseInt(e.target.value, 10) || 0 
```

---

## `??`「左が存在しないなら右を使う」（論理積）

「存在しない」= `null` か `undefined` だけ

```js
''   ?? 'システムテーマ'  // → ''    （空文字は「存在する」のでそのまま）
0    ?? 'システムテーマ'  // → 0     （0 も「存在する」のでそのまま）
null ?? 'システムテーマ'  // → 'システムテーマ'（null は存在しない）
```

> 「左の人が欠席のときだけ、右の人に代わってもらう。0点でも出席なら本人が答える。」

- `??` は `||` の「厳しすぎる判定」を改善するために後から作られた
- `0` や空文字も「ちゃんと存在する値」として扱いたい場面で使う

左辺が false の場合は左辺を返し、右辺を評価しない

```ts
// null/undefined のときだけデフォルト値を使う
getSavedTheme() ?? getSystemTheme()
```

---

## `&&`「左がOKなら右を実行する」

```js
true  && '実行する'  // → '実行する'（左がOKなので右を返す）
false && '実行する'  // → false     （左がダメなので止まる）
```

> 「関所。左を通れたら右に進める。通れなかったらそこで止まる。」

```tsx
// 条件が真のときだけ表示する
{condition && <Component />}
```

---

## 3つを並べて比べる

| 演算子 | 右を使う条件 | 「ダメ」の範囲 |
|---|---|---|
| `\|\|` | 左が falsy なとき | `0`, `''`, `null`, `undefined` など全部 |
| `??`   | 左が null/undefined のとき | `null`, `undefined` だけ |
| `&&`   | 左が truthy のとき（逆） | — |
