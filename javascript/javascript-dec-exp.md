# JavaScript 式と宣言

|    |  宣言  |  式  |
| ---- | ---- | ---- |
|  何をするか  |  名前をつけて登録する  |  値を生み出す  |
|  日常のたとえ  |  「田中さんという社員を登録します」  |  「この仕事をやってくれる人（無名でもOK）」  |

---

関数で見る具体例

```js
// ---- 関数宣言（Declaration） ----
function greet() {
  return "おはよう";
}

// ---- 関数式（Expression） ----
const greet = function () {
  return "おはよう";
};
```

一見すると「= があるかないか」に見えますが、本質はこうです。

- 関数宣言 → greet という名前の関数をそのスコープに登録する行為
- 関数式 → 関数という値を作って、それを変数に代入している

---

「値を生み出す」とはどういうことか

式は、どこかに書くとそこが値になります。

```js
// 数値リテラルも式（値を生み出す）
1 + 2           // → 3 という値になる

// 関数式も値を生み出す。だから変数に入れたり、引数に渡せる
const fn = function () { return "hello"; };  // 変数に入れる
setTimeout(function () { console.log("1秒後"); }, 1000);  // 引数に渡す
```

宣言は値を生み出さないので、こういう使い方はできません。

```js
// これはエラー！宣言は値ではないので代入できない
const fn = function greet() { ... };  // ← これは実は「名前つき関数式」であって宣言ではない

// 宣言は単独で書くもの
function greet() { ... }  // OK。これだけで完結する
```

---

もう一つの大きな違い：巻き上げ（hoisting）

```js
// 関数宣言は「巻き上げ」られるので、定義より前に呼べる
sayHello();  // ✅ "Hello!" と表示される
function sayHello() {
  console.log("Hello!");
}

// 関数式は変数への代入なので、定義より前には呼べない
sayHello();  // ❌ エラー！ まだ代入されていない
const sayHello = function () {
  console.log("Hello!");
};
```

宣言はファイルを読み込んだ瞬間に「登録」されるので、コードの上のほうでも使えます。

式はその行が実行されて初めて値が作られるので、順番通りにしか使えません。

---

核心は「名前を登録する行為か、値を生み出す行為か」の違いです。

|    |  宣言  |  式  |
| ---- | ---- | ---- |
|  本質  |  名前を登録する  |  値を生み出す  |
|  巻き上げ  |  される（定義前に使える）  |  されない  |
|  = の有無  |  たまたま = がないことが多い  |  たまたま = で受け取ることが多い  |
