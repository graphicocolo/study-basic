# JavaScript コールバック

「コールバック」= 「後で呼び返す（call back）」

具体例で理解する：  

```javascript
// ボタンがクリックされたら「こんにちは」と表示する                   
button.addEventListener('click', function() {
  console.log('こんにちは');
});
```

この `function() { console.log('こんにちは'); }` がコールバック関数。

なぜ「コールバック」と呼ぶのか：

1. あなたが関数を渡す（「これを後で実行してね」）
2. ボタンがクリックされるまで待機
3. クリックされたら、渡された関数が「呼び返される（call back）」
  
普通の関数との違い： 

- 普通の関数：自分で直接呼ぶ → sayHello()
- コールバック：誰かに渡して、条件が揃ったら呼んでもらう
  
要するに：「今すぐ実行しないけど、必要になったら実行してね」と渡す関数のこと。

|  部分  |  役割  |
| ---- | ---- |
|  button  |  イベントの発生元（対象要素）  |
|  'click'  |  監視するイベントの種類  |
|  function() { console.log('こんにちは'); }  |  イベントリスナー（コールバック関数）  |
|  console.log('こんにちは')  |  リスナーの中で実行される処理  |

イベントリスナーは `function() { ... }` という関数全体です。`console.log('こんにちは')` はその関数の中身（実行される処理）であって、リスナーそのものではありません。

イベントハンドラーは、イベントリスナーとほぼ同じ意味で使われます。Reactでは「イベントハンドラー」と呼ぶことが多い。addEventListenerでは「イベントリスナー」と呼ぶことが多い

日常で例えると

「玄関のチャイムが鳴ったら、『こんにちは』と言う」

- チャイム = 'click'（イベント）
- 「『こんにちは』と言う」という行動の決まり事 = function() { ... }（イベントリスナー）
- 実際に「こんにちは」と言うこと = console.log('こんにちは')（処理の実行）

登録しているのは**「こんにちはと言う行動の決まり事」（関数）**であって、「こんにちは」という発言そのものではない、という区別です。
